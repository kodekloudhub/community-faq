# KKE and 100 Days Challenges

This mini FAQ pertains to the task unlock logic for KodeKloud Engineer (KKE) tasks. 100 Days challenges (campaign topics) are part of the KKE platform therefore there are some similarities.

There are currenty two campaign topics:
* 100 Days of DevOps
* 100 Days of Cloud

Then there are the regular topics like "Linux Level 1" etc.

## Task Unlock Logic

The following describes in what order and how frequently you can attempt the tasks, which is dependent on your subscription plan.

A day from the point of view of this platform begins at 00:00 UTC (05:30 IST).

### Free and Standard Plans

**Unlock Logic**
* When a topic is added to **Active Practice, 1 task unlocks per day**
* Campaign topic tasks unlock **strictly in sequence**
* Regular topic tasks you can unlock in any order.
* Next task unlocks **only after the previous task is marked as completed**

**Attempt Restrictions**
* You can attempt **only 1 task per topic per day**. This can be broken down as
    * One task from each campaign topic, plus
    * One task from regular topics,
    * Making a maximum possible 3 tasks per day.
* Even if the next task is unlocked in a topic, you must **wait until the next day** to attempt it
* **No skipping** of campaign topic tasks allowed

### Pro and AI Plans

You can do unlimited number of tasks per day.

**Unlock Logic**
* When a topic is added to **Active Practice**, you can unlock **multiple tasks on the same day**
* Campaign topic tasks unlock **strictly in sequence**
* Regular topic tasks you can unlock in any order.
* Next task unlocks **only after the previous task is marked as completed**

**Attempt Restrictions**
* You can attempt **unlimited tasks per day**
* **No skipping** of campaign topic tasks allowed
